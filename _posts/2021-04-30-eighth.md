[전자서명]
: 디지털 시그니처, 디지털 데이터의 서명
: 서명된 메시지는 컴퓨터 네트워크를 통해 전송될 수 있고, 수신할 수 있음. 
(1) 인증 및 무결성 기능
: 신뢰를 제공하기 위해 디지털 데이터를 생성한 사람이 누구인지 인증이 필요하며, 변조되지 않았다는 무결성 기능을 제공해야함.
(2) 원본 데이터와 서명값을 묶어주는 기능
: 디지털 데이터는 데이터 값으로 존재하기 때문에 데이터 자체도 값으로 있고 서명 값도 값으로 있어 별도의 값으로 두 개가 존재함.
(3) 서명 구별 기능
: 원본에 사인을 했는데 그 문서를 복제했을 때 원본의 서명과 복제본의 서명을 식별할 수 있어야함.

[전자서명의 요구조건]
(1) 위조 불가
: 정당한 서명자만이 전자서명을 생성할 수 있어야 함.
(2) 서명자에 대한 인증
: 정당한 서명자가 생성했는지 빠른 확인이 가능해야 함.
(3) 재사용 불가
(4) 변경 불가
(5) 부인 방지
: 서명 사실을 부인하는 게 불가능해야 함.

[전자서명의 개념 정리]
: 비밀번호를 이용한다는 입장에서 동일한데, 검증 과정에서 쓰는 건 누구나 확인할 수 있도록 공개된 정보를 가지게 함.
: A와 B가 서로 데이터를 주고받고 이에 대해 무결성 값을 붙임으로써 해시함수를 이용해 검증을 함. 이는 A와 B가 키를 가지고 있다고 할 수 있음.
: 이 키를 공개 정보라고 하고, 공개키라고도 부름.
: 개인키 – 개개인이 자신만 알고 있는 정보, 개인키를 가지고 서명을 함
: 공개키 – 모든 사람에게 공개하는 정보, 누구나 쉽게 취득이 가능하고 공개키를 가지고 서명을 검증함.

[서명의 구성 요소]
(1) P: 원본 데이터 집합
(2) A: 서명값 집합
(3) K: 가능한 키 집합
(4) S: 서명 생성의 집합
(5) V: 서명 검증 집합

[전자서명의 구성]
: 서명을 하는 행위 - 키를 만들고 서명 알고리즘에 키를 주입시켜 서명값을 생성
1) 키 생성: 개인키와 공개키를 얻음. 
2) 서명 생성: 메시지와 개인키가 서명 생성이라는 알고리즘의 입력값으로 작용하고 출력으로 서명값이 나옴.
3) 서명 검증: 입력값으로 메시지, 서명값, 공개키를 넣고 검증 성공 또는 실패라는 출력을 내놓음.

[RSA 암호]
: 지수 연산을 통해 서명 생성을 하고, 지수 연산을 통해 서명 검증 과정을 수행하는 방법
: 서명 생성과 검증 과정을 뒤집으면 암복호화 과정이 될 수 있음.
- 암호화: 메시지를 감추는 것, 공개키
- 복호화: 감춰진 메시지를 복구해서 내용을 열람하는 행위, 개인키
: 2000비트 이상을 쓰게 돼 계산량이 많이 들고 데이터가 큼.
: 메시지 크기만큼 서명값도 커지게 되기에 메시지를 아무거나 넣을 수 없음.

[메시지 부가형 전자서명]
: 현재 가장 대표적인 전자서명 기법
: 임의의 길이 메시지에 대한 서명이 가능함. 
: 메시지 길이와 상관없이 고정된 길이의 서명값을 가짐. 
: 서명값 거대화를 방지하기 위해 메시지를 해시한 다음에 해시값으로 해시값의 서명을 생성하는 방법
: 대표적으로 RSA 전자서명, 타원곡선 전자서명

[RSA 전자서명]
: 인수문제 기반
: 공개키(n, e), 개인키(p, d, q)

[타원곡선 전자서명]
: RSA가 점점 계산상으로 효율성이 떨어진다고 생각해 그에 대비해 만들어진 암호
: 타원곡선 이산대수문제 기반
: 공개키(P, Q), 개인키(d)

[비트코인과 전자서명]
: 공개키를 가지고 계좌를 생성하고 개인키로 계좌의 소유권을 나타내며 거래를 승인함.

[거래의 승인과 검증]
: 개인키를 가지고 서명이라는 행위로 거래를 승인하고 공개키를 가지고 검증 과정을 진행함.
: 인증과 무결성을 굉장히 효율적으로 제공하는 전자 서명을 거래의 승인과 검증에 적용.
: 개인키를 특정 개인과 연결하기 위해서는 신뢰할 수 있는 제3의 기관등이 필요함.
